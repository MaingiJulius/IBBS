# SLIDE 2: Introduction & Problem
*   **The Issue:** Manual booking systems are slow, insecure, and opaque.
*   **The Solution:** IBBS - A web-based "E-Ticketing" platform for Wema Travellers.
*   **Key Benefit:** Efficiency, Transparency, and Convenience.

---

# SLIDE 3: System Objectives
1.  **Automation:** Remove manual paperwork.
2.  **Accessibility:** Allow booking from anywhere.
3.  **Control:** Give admins full oversight of the fleet.
4.  **Insight:** Provide real-time financial reporting.

---

# SLIDE 4: Technologies Used
*   **Frontend:** HTML5, CSS3, JavaScript (Responsive Design).
*   **Backend:** PHP 8.0+ (Server-side logic).
*   **Database:** MySQL (Relational Data Management).
*   **Server:** Apache (XAMPP Environment).

---

# SLIDE 5: System Architecture
*   **3-Tier Architecture:** Client (Browser) <-> Server (PHP) <-> Database (MySQL).
*   **Security:** 
    *   Input Validation (Prevent SQL Injection).
    *   Session Management (Prevent unauthorized access).
    *   Encryption (Password Hashing).
