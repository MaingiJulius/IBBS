# SLIDE 2: Introduction
*   **Overview:** A web-based platform to automate bus ticket booking.
*   **Current State:** Manual booking (queues, paper tickets).
*   **Desired State:** Digital booking (online, instant, secure).

---

# SLIDE 3: Problem Statement
*   **Time Consuming:** Passengers queue for hours.
*   **Inefficient:** Manual records are prone to error.
*   **Revenue Leakage:** Hard to track cash payments manually.
*   **Limited Access:** Booking is only possible during office hours.

---

# SLIDE 4: Objectives
*   **Main Goal:** Build a robust Online Bus Booking System.
*   **Specific Goals:**
    1.  Enable 24/7 online seat reservation.
    2.  Manage Fleet (Buses) and Routes dynamically.
    3.  Generate real-time financial and occupancy reports.
    4.  Secure access via Role-Based Authentication (Admin, Agent, User).

---

# SLIDE 5: Methodology

    *   Backend: PHP
    *   Database: MySQL
*   **Tools:** VS Code, XAMPP, Git.
